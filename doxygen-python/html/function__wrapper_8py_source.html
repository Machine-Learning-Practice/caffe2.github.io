437 # Note [field_name versus name]
438 # ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
439 # What is the difference between "field_name" and "name"?
440 #
441 # Return values of ATen operators always have a name: if it is not
442 # explicitly assigned a name inside native_functions.yaml like func:
443 # myop() -> (Tensor indices, Tensor value), then the codegen will
444 # automatically assign it a name like result0, or name might be
445 # specified inside Declarations.cwrap. We don't want these assigned
446 # names to become part of the public API when we return a namedtuple for
447 # any such multiple-return function.
448 #
449 # Thus field_name is like name, but it is defined only when there is a
450 # name specified in native_functions.yaml. If field_name is defined,
451 # then the codegen would generate code to return namedtuple. Otherwise,
452 # it would just return tuple.
